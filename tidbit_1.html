tidbit 1: The Locus of a Declaration

Consider the following code(from basic.scope.pdecl#1):

unsigned char x = 12;
{ unsigned char x = x; }

This code exhibits undefined behavior because of the following two rules:

  • The locus of a declaration that is a declarator is immediately after the complete declarator .
  • A program point P is said to follow any declaration in the same translation unit whose locus is before P.

In the example above, when initializing the second x, name lookup also includes the second x itself. By the rules of C++ name lookup, this x is considered a better match, but it is not yet initialized. Therefore, using it in its own initializer leads to undefined behavior.

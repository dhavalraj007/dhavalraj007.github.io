tidbit 2: User Provided Functions

User Provided = User declared AND (not defaulted on first declaration) AND (not deleted on first declaration)

for the function to be considered provided by user ,

  • of course the user has to declare it and not the language.
  • lets say the user has declared it, then if the user =defaults the definition then it is again the language that provides the definition and thus it is not User provided anymore.
  • if the user declares it and says it is deleted , then it is not even provided by anyone not even the language.
  • this defaulted and deleted things apply only at first declaration level (which is the type creation stage in the header file). if user only declares the function at the first declaration and then (maybe in the source file) defines as =default then it is also called to be provided by User and not the language(although the definition is provided by the language, just not at the type creation stage). =delete is not even allowed after the first declaration.

from cppreference: https://en.cppreference.com/w/cpp/language/function.html#User-provided_functions

// All special member functions of “trivial” are
    // defaulted on their first declarations respectively,
    // they are not user-provided
    struct trivial
    {
        trivial() = default;
        trivial(const trivial&) = default;
        trivial(trivial&&) = default;
        trivial& operator=(const trivial&) = default;
        trivial& operator=(trivial&&) = default;
        ~trivial() = default;
    };
    
    struct nontrivial
    {
        nontrivial(); // first declaration
    };
    
    // not defaulted on the first declaration,
    // it is user-provided and is defined here
    nontrivial::nontrivial() = default;
    // nontrivial::nontrivial() = delete;  // not allowed
